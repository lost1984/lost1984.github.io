# 日报任务兜底流程

适用对象：早间日报 / 晚间日报 / 类似“检索→整理→建文档→回链路”的固定任务。

## 原则
- 先闭环，再解释
- 小错误先自救，不直接退回用户
- 只在缺权限、缺关键目标、或高风险动作时再打断用户

## 固定执行顺序
1. **先产出正文**
   - 能直接拼 Markdown，就不要依赖 Python
   - 若要跑脚本：先试 `python3`，不要默认 `python`
   - 若 `python3` 也不通，再改用 shell / node / 直接文本写文件

2. **再确认文档链路**
   - 优先尝试创建飞书文档
   - 若无法更新旧文档，先新建文档拿到可分享链接
   - 不要因为“索引表暂时没补”就让整单失败

3. **索引补录分级处理**
   - 能补：继续补
   - 暂时不能补：先把正文文档建出来，并在回复里明确“索引待补”
   - 不允许因为索引未补而整单回退

4. **最后发回执**
   - 至少给出：标题、链接、关键词、1句主判断
   - 用户应拿到可点开的最终产物，而不是过程解释

## 常见故障兜底
### A. `python` 不存在
- 先试：`python3`
- 不行：改用 `node`
- 再不行：直接写 Markdown 文件 / 直接调用文档工具

### B. 文档更新工具不可用或目标不明确
- 先新建独立日报文档
- 保住正文和链接
- 后续再补索引/回填

### C. 检索结果不足
- 先保留已验证来源
- 降低条目数量，不凑数
- 保证“少而真”优先于“多而虚”

## 禁止行为
- 因为一个命令失败就直接回复“环境不支持”
- 未验证就说“没有工具”
- 没有尝试替代方案就把任务退回
- 产物未落地就只交付口头框架

## 最低交付标准
一次日报任务至少应完成其中前两项：
- [x] 有正文
- [x] 有文档链接
- [ ] 有索引补录（做不到可延后，但不能阻塞前两项）
